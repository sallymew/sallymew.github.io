es6-learning

发表于 2016-01-13   |   分类于 前端   |  

前言

ECMAScript 6（以下简称ES6）是javascript语言的下一代标准，已经在2015年6月正式发布了。他的目标是使得javascript语言可以用来编写复杂的大型应用程序，成为企业级开发语言。因为这个版本是2015年发布的，因此也成为ECMAScript 2015。

使用ES6

目前各大浏览器的最新版本对ES6的支持情况，可以查看http://kangax.github.io/compat-table/es6/。

使用chrome

启用实验性javascript

使用转码器-babel

babel是一个广泛使用的ES6转码器，可以将ES6代码转为ES5代码，从而在浏览器或者其他环境中执行。

命令行环境下，可以通过按照babel-cli和babel-preset-es2015包使用es6特性。

babel自带babel-node命令，可以支持node的repl环境的所有功能，可以直接运行es6代码。

使用node

nodejs是javascript语言的服务器运行环境，对ES6的支持度比浏览器更高，通过node，可以体验更多的ES6特性。

使用node运行ES6代码时，有时需要使用严格模式use strict，具体可以参考：ECMAScript 2015 (ES6) in Node.js

有了工具就可以开启ES6新特性的学习之旅啦~↖(^ω^)↗

变量声明

ES5中变量的声明方式只有2种：var和function，ES6又添加了let、const、class、import/export（跨模块变量声明）4种方式。

let命令

ES6新增let命令，用来声明变量。它的用法类似var，但是所声明的变量只在let命令所在的代码块内有效。

块级作用域

ES6规定，使用let和const声明的变量都是只在声明所在的块级作用域内有效，var声明的变量仍然具有全局作用域，函数本身的作用域也是其所在块级作用域内有效。

使用var声明时，i在全局范围内有效，使用let声明时，i只在本轮循环有效。

使用babel进行代码转换时，可以发现：

这也是我们解决闭包问题时的一个好方法~

ES5只有全局作用域和函数作用域，没有块级作用域，这带来很多不合理的场景：

  1. 内层变量覆盖外层变量
  2. 用于计数的循环变量泄露为全局变量

有了块级作用域，外层作用域无法读取内层作用域的变量，内层作用域可以定义外层作用域的同名变量，也使得IIFE不再必要了。

不存在变量提升

let不像var会发生变量提升现象，所以，使用let声明的变量一定要先声明后使用，否则报错。

特殊的，在区块中使用let命令声明变量时，它声明的变量就绑定了这个区域，不再受外部的影响。

ES6明确规定，如果区块中存在使用let和const命令声明的变量，那么这些变量从一开始就形成了封闭作用域，凡是在声明之前就使用这些命令都会报错。在语法上，称为暂时性死区（Temporal dead zone，简称TDZ）。

ES6中规定暂时性死区和不存在变量提升，主要是为了减少运行时错误，防止在变量声明前就使用这个变量，从而导致意料之外的行为。

不允许重复声明

let不允许在相同作用域内，重复声明同一个变量。

const命令

基本用法

const命令用于声明常量，一旦声明，常量的值就不能被改变。

常规模式下，对const常量赋值虽然无效但是不会报错，严格模式下会报错。const常量只声明未赋值常规模式下不报错，严格模式下SyntaxError: Unexpected token ;。

特性

const命令声明的变量与let声明的变量类似：

  • 只在声明所在的块级作用域内有效
  • 不存在变量提升，存在暂时性死区
  • 不可重复声明

对于复合类型的变量，由于变量名不指向数据，而是指向数据所在的地址。const命令只是保证变量名的地址不变，但是并不能保证该地址的数据不变。

所以将一个对象（包括数组）声明为常量必须十分小心。

全局对象的属性

全局对象是最顶层的对象，在浏览器环境是指window对象，在nodejs环境指的是global对象。ES5中，全局对象的属性和全局变量是等价的。这种规定被视为javascript语言的一大问题，因为很不容易不知不觉就创建了全局变量。ES6为了改变这一点，一方面规定，var命令和function命令声明的全局变量依旧是全局对象的属性，另一方面规定，let命令、const命令和class命令创建的全局变量，不属于全局对象的属性。

解构赋值

解构赋值（destructuring assignment）句法是一个Javascript表达式，通过这个表达式，就可以从array或者object中通过一种句法来提取数据。这个句法与array的构造和object的字面定义对应。

对象字面量和数组字面量提供了一种简单的定义一个特定的数据组的方法，一旦创建了这类数据组，就可以用任意方法使用这些数据组。

解构数组

基本用法

用途

  1. 交换变量

    var x = 1, y = 2;
// ES5
x = [y, y=x][0];
// ES6
[x, y] = [y, x];

  2. 函数返回多值

    function f() {
    return [1, 2];
}
var [a, b] = f();
a // 1
b // 2
var c = f();
c // [1, 2]

解构字符串

字符串也可以解构赋值，这是因为此时的字符串被转换成了一个类似数组的对象

类似数组的对象都有一个length属性，因此可以对这个属性解构赋值。

可遍历结构

但是，当右边不是可遍历的结构时，将会报错。事实上，只要某种数据解构具有Iterator接口，都可以采用数组形式的解构赋值。

其他

解构赋值时，如果等号右边是数值和布尔值，则会先转为对象。

解构赋值的规则是，只要等号右边的值不是对象，就先将其转化为对象。由于undefined和null无法转为对象，所以对其进行解构赋值时会出现TypeError。

本质上，解构赋值这种写法属于模式匹配，只要等号两边的模式相同，左边的变量就会被赋予相应的值。如果解构不成功，变量的值为undefined。当等号左边的模式只匹配一部分的等号右边的数组，这种情况下解构依然成功。
