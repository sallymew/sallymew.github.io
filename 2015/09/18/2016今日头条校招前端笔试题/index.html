闭包

定义

闭包就是由函数创造的一个词法作用域，里面创建的变量被引用后，可以在这个词法环境之外自由使用。简单来说，闭包就是能够读取其他函数内部变量的函数。在javascript语言中，只有父函数内部的子函数可以读取父函数的局部变量。因此可以把闭包简单理解成–定义在一个函数内部的函数。在本质上，闭包就是将函数内部和函数外部连接起来的一座桥梁。这就是javascript语言特有的链式作用域结构，子对象会一级一级地向上寻找所有父对象的变量。所以，父对象的所有变量，对子对象都是可见的，反之则不成立。内部函数一直拥有从父函数作用域取得的变量，这些值不会随着父函数的执行完成而被销毁。Segment Fault上关于闭包的总结

作用

闭包通常被用来创建内部变量，使得这些变量不能被外部对象随意修改，同时又可以通过指定的函数接口来操作

特点

  1. 作为一个函数变量的一个引用，当函数返回时，其处于激活状态

  2. 一个闭包就是当一个函数返回时，一个没有释放资源的栈区

原题

闭合浮动

如何消除子元素浮动带来的父元素的高度塌陷–这是一个典型的闭合浮动的问题。清除浮动对应着CSS中的属性是 clear：none|left|right|both ，而闭合浮动是使浮动元素闭合，从而减少浮动带来的影响。

清除浮动的2种方法：

  1. 在浮动元素末尾添加空元素，设置clear: both

  2. 触发BFC，IE下还要触发hasLayout

BFC

块级格式化上下文，可以看做是隔离了的独立容器

  1. 如何触发BFC：

    1. 根元素默认拥有BFC特性

    2. float: left|right

    3. overflow不为visible: overflow: hidden|auto|scroll

    4. display: table-cell|table-caption|inline-block

    5. position: absolute|fixed

  2. 作用：

    • 阻止父子元素的外边距margin折叠

    • 不会重叠浮动元素

    • 可以包含浮动元素

  3. 通俗的来说，触发了BFC的元素就是一个独立的盒子，里面的子元素不会在布局上影响外面的元素，反之亦然，同时，BFC仍属于普通流

hasLayout

在IE6-7中显示引擎使用的是布局layout这个概念，当外面说一个元素拥有layout的时候，是指其hasLayout属性为true，拥有布局的元素负责本身和其子元素的尺寸设置和定位

  1. position: absolute

  2. float: left|right

  3. display: inline-block

  4. width除auto以外的任意值

  5. height除auto以外的任意值，如height: 1%

  6. zoom除normal以外的任意值

  7. IE7中设置 overflow: hidden|auto|scroll

清除浮动的代码

方法1

.clearfix:after {
    clear: both; 
    content: "."; /*Firefox中为空仍然会产生额外的空隙，但是此值为空时不需要height：0和visibility：hidden*/
    display: block; 
    height: 0; /*避免生成内容对原有布局的高度产生影响*/
    font-size: 0;
    line-height: 0;
    visibility: hidden;
}
.clearfix {*zoom: 1;}/*清除ie6-7下的浮动，作用是触发hasLayout*/

方法2：精益求精清除浮动    
/**
* 在标准浏览器下使用
* 1 content内容为空格用于修改opera下文档中出现contentEditable属性时在清理浮动元素上下的空白
* 2 使用display:table可以防止容器和子元素top-margin折叠，这样能使清理效果与BFC，IE6/7下的zoom：1一致

基本原理考察

从输入URL到整个网页加载完毕及显示在屏幕上的整个流程

  1. 浏览器检测缓存，若缓存存在且足够新鲜则提供给客户端，否则组装一个HTTP报文

  2. 浏览器会利用DNS服务器查询到URL域名对应的IP地址，然后通过三次握手建立TCP连接之后，浏览器作为连接的发起方，会用服务器发送一个HTTP请求报文

  3. 服务器会一直监听HTTP所在的80号端口，在发现上面的请求之后，首先检测是否需要验证，无需验证或验证完毕后便会向浏览器发送HTTP响应报文，内容是URL对应的资源内容

  4. 浏览器收到来自服务器的响应之后，会根据不同的响应状态码（200 ok，403 forbidden， 404 not found）做出不同的处理，会使用渲染引擎将HTML页面、CSS源代码生成展示页面，同时下载页面中的JS、图片等文件并解析

HTTP状态码小结

HTTP1.0和1.1的区别：keepalive

  • 1XX 信息

  • 2XX 成功：200 ok

  • 3XX 重定向：301 永久重定向，302 临时重定向（ajax获取不到，见下面解释部分/HTTP1.1中302为Found，但也是一个别的地址），303 see other，304 not modified，原来缓存的文档还可以继续使用，未过期

  • 4XX 客户端错误：400 bad request，401 unauthorized需要用户名和密码，403 forbidden，404 not found

  • 5XX 服务器错误：500 Internal Server Error
