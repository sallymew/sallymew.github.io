ES5中的类和继承

Javascript是一种面向原型的语言，函数是javascript的一等公民，并在ECMAScript5中，能够模拟类和继承。

定义类

在javascript中，灵活的函数允许我们模拟常用于处理类的封装。通过这个技巧，能够扩展对象的原型。

var Animal = (function() {
    function Animal(name) {
        this.name = name;
    }

    Animal.prototype.doSomething = function() {
        console.log("I'm a " + this.name);
    };

    return Animal;
})();

var lion = new Animal("Lion");
lion.doSomething();

上面的代码中，定义了一个带有属性和方法的类，这个构造器通过函数（匿名函数Animal）定义，并能够实例化属性。通过使用原型，可定义被实例化的函数。而问题是，假设你了解原型继承和一些基于类继承的语言，你会感到很混乱。奇怪的是，javascript有class关键字，却不能做任何事，而现在ECMAScript 6用上了~

原型继承

var ClassA = function() {
    this.name = "class A";
}
ClassA.prototype.print = function() {
    console.log(this.name);
}
var a = new ClassA();
a.print();

var inheritFrom = function(child, parent) {
    child.prototype = Object.create(parent.prototype);
}

var ClassB = function() {
    this.name = "class B";
    this.surname = "I'm the child";
}
inheritFrom(ClassB, ClassA);
ClassB.prototype.print = function() {
    ClassA.prototype.print.call(this);
    console.log(this.surname);
}
var b = new ClassB();
b.print();

原型链：javascript中，每个构造函数都有一个prototype属性，指向另一个对象。这个对象的所有属性和方法，都会被构造函数的实例继承。这意味着，我们可以把那些不变的属性和方法直接定义在prototype对象上。在进行属性查找是，javascript会向上遍历原型链，直到找到给定名称的属性为止，到查找到达原型链的顶部，也就是Object.prototype。可以使用hasOwnProperty来判断对象是否包含该属性，使用in来判断对象和其原型链中是否包含。下面是一个完整的继承的实例：

function Person(){} //空的构造函数
Person.prototype = {
    init: function(name){
        this.name = name;
    },
    getName: function(){
        return this.name;
    }
    call: function(){
        alert("I am a person");
    }
}
function Employee(){}
Employee.prototype = new Person();
Employee.prototype.constructor = Employee; //显示指定constructor可以防止prototype改写带来的问题

ES6中的类和继承

简单的ES6类

class AnimalES6 {
    constructor(name) {
        this.name = name;
    }

    doSomething() {
        console.log("I'm a " + this.name);
    }
}

var lionES6 = new AnimalES6("Lion");
lionES6.doSomething();

尽管运行结果和前面的一样，但是对于习惯写类的开发者来说，更易编写和阅读，因为这不需要原型，并且能够使用constructor关键字定义构造器。此外，ES6的类引入了一些ES5所没有的语法，例如，必须通过new调用构造函数，或者不能用new尝试去构造类方法，另一个改变是，方法都是不可枚举的（存疑？）。有趣的是，两个版本可同时共存使用。归根结底，即使用new关键字实例化一个添加在原型的函数，“方法”在这里仅仅是对象的一个函数属性。另一个基于类开发的核心特性是，getter和setter都在ES6中得到支持，这使得方法的用途变得更加明显：

ES6类包含私有属性–symbol

class  AnimalES6 {
    constructor(name) {
        this.name = name;
        this._age = 0;
    }

    get age() {
        return this._age;
    }

    set age(value) {
        if(value < 0) {
            console.log("We do not support undead animals");
        } else {
            this._age = value;
        }
    }

    doSomething() {
        console.log("I'm a " + this.name);
    }
}

var lionES6 = new AnimalES6("Lion");
lionES6.doSomething();
lionES6.age = 5;

这样写很方便，但是会看到一个javascript的常规警告：“不是真正私有的”私有成员（_age）。创建私有成员可以使用closure space

var createProperty = function(obj, prop) {
    var currentValue = obj[prop];
    Object.defineProperty(obj, prop, {
        get: function() { return currentValue;},
        set: function() { currentValue = value;},
        enumerable: true,
        configurable: true
    });
}

var entity = {
    property: "hello world"
}
createProperty(entity, "property")

与一般的get/set方式相比，这种方式貌似性能更优。不过，现在有一个更好的、新的ECMAScript6特性来完成这个事情：symbols

var ageSymbol = Symbol();

class  AnimalES6 {
    constructor(name) {
        this.name = name;
        this[ageSymbol] = 0;
    }

    get age() {
        return this[ageSymbol];
    }

    set age(value) {
        if(value < 0) {
            console.log("We do not support undead animals");
        } else {
            this[ageSymbol] = value;
        }
    }

    doSomething() {
        console.log("I'm a " + this.name);
    }
}

var lionES6 = new AnimalES6("Lion");
lionES6.doSomething();
lionES6.age = 5;

那么，什么是symbol呢？他是唯一且不可改变的数据类型，可用于标识对象属性。如果没有symbol，是不能获取该属性的。所以，这是一个可创建更私有的成员属性途径。或者，至少没那么容易访问到。symbol对于创建唯一的名字是很有用的，但唯一性不意味着私有性，唯一性只意味着，为一把钥匙添加一个新符号后，当你需要它的适合，肯定不会被其他钥匙所干扰。但这还不算真正的私有，因为通过Obje.getOwnPropertySymbols, downstream consumers能获取symbol属性。

继承

var legsCountSymbol = Symbols();
class InsectES6 extends AnimalES6 {
    constructor(name) {
        super(name);
        this[legsCountSymbol] = 0;
    }

    get legsCount() {
        return this[legsCountSymbol];
    }

    set legsCount(value) {
        if (value < 0) {
            console.log("We do not support nether or interstellar insects");
        } else {
            this[legsCountSymbol] = value;
        }
    }

    doSomething() {
        super.doSomething();
        console.log("And I have " + this[legsCountSymbol] + "legs!");
    }
}

var spiderES6 = new InsectES6("Spider");
spiderES6.legsCount = 8;
spiderES6.doSomething();

多亏了extends关键字，它能让你继承类而产生子类，并能使用super关键字保持对根类的引用。在这些新增且强大的特性下，无需再通过原型来创建类和实现继承了。
